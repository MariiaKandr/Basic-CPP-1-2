## Создание разных видов конструкторов в C++. 
Конструктор — это специальный метод, который автоматически вызывается при создании объекта и инициализирует его поля.

---

### 1. Виды конструкторов

#### Конструктор по умолчанию (без параметров)
Вызывается, когда объект создаётся без аргументов. Если вы не напишете ни одного конструктора, компилятор создаст свой (пустой), но лучше написать явно.

```cpp
class Student {
    string name;
    int id;
public:
    Student() {                     // конструктор по умолчанию
        name = "Неизвестно";
        id = 0;
    }
};
```

#### Конструктор с параметрами
Позволяет задать начальные значения прямо при создании объекта.

```cpp
Student(string n, int i) {
    name = n;
    id = i;
}
// Использование:
Student s("Анна", 123);
```

#### Конструктор копирования
Создаёт новый объект как копию существующего. Принимает ссылку на другой объект того же класса. Если его не определить, компилятор сделает «поверхностное копирование» (копирует все поля побайтово). Для классов с динамической памятью (указатели) нужно писать свой конструктор копирования для **глубокого копирования**.

```cpp
Student(const Student& other) {
    name = other.name;
    id = other.id;
    // если есть указатель на динамическую память, нужно выделить новую и скопировать данные
}
```

### 2. Список инициализации (member initializer list)

Пишется после `:` перед телом конструктора. Используется для:
- инициализации константных полей (`const`),
- инициализации полей-ссылок,
- инициализации полей, у которых нет конструктора по умолчанию,
- более эффективной инициализации (вызывается сразу, а не сначала создаётся пустое поле, потом присваивается).

```cpp
class Book {
    const string title;   // константное поле
    int pages;
public:
    Book(string t, int p) : title(t), pages(p) { }
    // title инициализируется сразу, а не в теле конструктора
};
```

### 3. Ключевое слово `explicit`

Запрещает неявные преобразования типов. Применяется к конструкторам с одним параметром. Без `explicit` компилятор может автоматически преобразовать, например, `int` в ваш объект, что может привести к неожиданным вызовам. С `explicit` такое преобразование будет только по явному запросу.

```cpp
class Integer {
    int value;
public:
    explicit Integer(int v) : value(v) {}
};

void print(Integer i) { ... }

int main() {
    Integer a = 10;   // ОШИБКА, explicit запрещает неявное преобразование
    Integer b(10);    // ОК
    print(20);        // ОШИБКА
    print(Integer(20)); // ОК
}
```

### Конструктор перемещения (Move Constructor)

Появился в C++11. Позволяет **переместить** ресурсы из временного объекта в новый, избегая дорогостоящего копирования. Принимает **rvalue-ссылку** (`&&`). После перемещения исходный объект остаётся в корректном, но неопределённом состоянии (обычно его больше не используют).

```cpp
class DynamicArray {
    int* data;
    int size;
public:
    // Конструктор перемещения
    DynamicArray(DynamicArray&& other) noexcept 
        : data(other.data), size(other.size) {
        other.data = nullptr;   // обнуляем, чтобы деструктор не удалил данные
        other.size = 0;
        cout << "Вызван конструктор перемещения\n";
    }
};

// Использование:
DynamicArray createArray() {
    DynamicArray temp(1000);
    return temp;   // вызывает конструктор перемещения (или RVO)
}
```

**Полезен** при возврате больших объектов из функций, при использовании `std::move`, при работе с контейнерами STL.

### Делегирующий конструктор (Delegating Constructor)

Позволяет одному конструктору вызывать другой конструктор того же класса для выполнения общей инициализации. Избегает дублирования кода.

```cpp
class Logger {
    string file;
    int level;
public:
    // Главный конструктор (с тремя параметрами)
    Logger(string f, int l, bool init) : file(f), level(l) {
        cout << "Главный конструктор\n";
    }

    // Делегирует трёхпараметрическому
    Logger(string f, int l) : Logger(f, l, true) { }

    // Делегирует конструктору с двумя параметрами
    Logger(string f) : Logger(f, 1) { }

    // Делегирует конструктору с одним параметром
    Logger() : Logger("log.txt") { }
};
```

**Правила:**  
- Вызов делегирования должен быть единственным в списке инициализации.  
- Нельзя одновременно инициализировать поля и делегировать.  
- Цепочка делегирования должна заканчиваться конструктором, который не делегирует.

## Деструктор

Деструктор — это специальный метод класса, который автоматически вызывается при уничтожении объекта. Его основная задача — освободить ресурсы, которые объект захватил за время жизни (память, файловые дескрипторы, сетевые соединения и т.п.).

### Синтаксис и особенности

- Имя деструктора совпадает с именем класса, перед которым стоит тильда `~`.
- Не принимает аргументов и не возвращает значения (даже `void`).
- Не может быть перегружен — у класса только один деструктор.
- Если вы не определите деструктор, компилятор создаст его автоматически (пустой).

```cpp
class Resource {
    int* data;
public:
    Resource() {
        data = new int[100];
    }
    ~Resource() {
        delete[] data;   // освобождаем память
        data = nullptr;  // хороший тон
    }
};
```

### Когда вызывается деструктор

- Для статических объектов — при выходе из области видимости.
- Для динамических объектов — при вызове `delete`.
- Для временных объектов — в конце полного выражения.

```cpp
void example() {
    Resource r;          // создаётся
    Resource* p = new Resource(); // создаётся
    // ...
}                        // r уничтожается автоматически
                         // p не удалён — утечка памяти!
delete p;                // только теперь деструктор для *p вызовется
```

### Виртуальный деструктор

Если класс предназначен для наследования (имеет виртуальные методы), **деструктор базового класса должен быть виртуальным**. Иначе при удалении объекта производного класса через указатель на базовый класс вызовется только деструктор базового класса, а ресурсы производного не будут освобождены.

```cpp
class Base {
public:
    virtual ~Base() { }   // виртуальный деструктор
};

class Derived : public Base {
    int* buffer;
public:
    Derived() { buffer = new int[100]; }
    ~Derived() { delete[] buffer; } // теперь вызовется
};

int main() {
    Base* p = new Derived();
    delete p;   // без virtual ~Base() вызвался бы только ~Base() — утечка
}
```

### RAII (Resource Acquisition Is Initialization)

Деструктор — ключевой элемент идиомы RAII: ресурс захватывается в конструкторе, освобождается в деструкторе. Это гарантирует отсутствие утечек даже при исключениях.

```cpp
class FileHandler {
    FILE* f;
public:
    FileHandler(const char* name) : f(fopen(name, "r")) { }
    ~FileHandler() { if (f) fclose(f); }
};
```


### Связь с конструктором перемещения и делегирующим конструктором

- **Конструктор перемещения** позволяет эффективно передавать владение ресурсом между объектами. После перемещения перемещаемый объект остаётся в состоянии, пригодном для уничтожения (деструктор обнуляет указатели).
- **Делегирующие конструкторы** помогают избежать дублирования кода инициализации; деструктор же остаётся единственным и отвечает за освобождение ресурсов независимо от того, какой конструктор создал объект.
