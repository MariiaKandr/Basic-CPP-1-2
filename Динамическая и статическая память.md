##  Создание объектов в **куче** (динамически), удаление их и отслеживание **жизненного цикла** объектов.

---

### 1. Статическое vs динамическое создание объектов

| **Статическое (на стеке)** | **Динамическое (в куче)** |
|----------------------------|---------------------------|
| `ClassName obj;`           | `ClassName* ptr = new ClassName;` |
| Память выделяется при входе в блок, освобождается автоматически при выходе. | Память выделяется оператором `new`, освобождается вручную `delete`. |
| Быстро, но размер и время жизни ограничены областью видимости. | Гибко, но требует явного удаления, иначе утечка памяти. |

```cpp
void example() {
    Point p1(1,2);          // статический объект
    Point* p2 = new Point(3,4); // динамический объект
    p2->display();
    delete p2;              // обязательно удалить!
} // p1 автоматически уничтожится здесь
```

---

### 2. Операторы `new` и `delete`

- **`new`** – выделяет память в куче и вызывает конструктор.
- **`delete`** – вызывает деструктор и освобождает память.
- Для массивов: `new[]` и `delete[]`.

```cpp
// Одиночный объект
User* u = new User("Alice", 25);
u->introduce();
delete u;

// Массив объектов
Item* arr = new Item[10];   // 10 раз вызывается конструктор по умолчанию
arr[0].setId(1);
delete[] arr;               // обязательно delete[]!
```

---

### 3. Жизненный цикл объекта

1. **Выделение памяти** (стек или куча)
2. **Инициализация** (вызов конструктора)
3. **Использование** (вызов методов)
4. **Уничтожение** (вызов деструктора)
5. **Освобождение памяти**

Для статических объектов этапы 1 и 5 – автоматические.  
Для динамических – программист управляет через `new`/`delete`.

```cpp
class Tracker {
public:
    Tracker() { cout << "Создан\n"; }
    ~Tracker() { cout << "Уничтожен\n"; }
};

void func() {
    Tracker t;          // создаётся здесь
}                       // уничтожается здесь

int main() {
    Tracker* t = new Tracker();  // создаётся
    delete t;                     // уничтожается
}
```

---

### 4. Утечки памяти и как их избежать

**Утечка** – когда память выделена, но не освобождена (указатель потерян или не вызван `delete`).

```cpp
void leak() {
    int* p = new int(5);
    // забыли delete p;
} // p исчезает, память утекла
```

**Правила:**
- На каждый `new` должен быть ровно один `delete`.
- На `new[]` – `delete[]`.
- После `delete` указатель лучше обнулить (`nullptr`), чтобы не стало «висячим».
- Используйте **RAII**: захват ресурса в конструкторе, освобождение в деструкторе (как в `FileHandler`).

---

### 5. Правило трёх (Rule of Three)

Если класс управляет динамической памятью (содержит указатель на выделенную память), нужно **явно** определить:
- **деструктор** – чтобы освободить память
- **конструктор копирования** – чтобы правильно скопировать данные (глубокая копия)
- **оператор присваивания** – чтобы корректно присвоить

Без этого компилятор сгенерирует поверхностное копирование (скопирует только указатель), что приведёт к двойному удалению и ошибкам.

```cpp
class IntArray {
    int* data;
    size_t size;
public:
    IntArray(size_t n) : size(n) {
        data = new int[n];
    }
    ~IntArray() { delete[] data; }
    IntArray(const IntArray& other) : size(other.size) {
        data = new int[size];
        for (size_t i = 0; i < size; ++i) data[i] = other.data[i];
    }
    // ... оператор присваивания тоже нужен
};
```
Удачи на практике! Внимательно разберите примеры из методички – они хорошо иллюстрируют каждый важный момент.
